## Workflow de Contribution

### 1. Récupérer les dernières modifications

```bash
git checkout main
git pull origin main
```

### 2. Créer une branche de travail

```bash
git checkout -b feature/nom-de-la-tache
```

Exemples :

```bash
git checkout -b feature/navbar
git checkout -b feature/page-accueil
git checkout -b feature/footer
git checkout -b feature/contact
```

### 3. Développer la fonctionnalité

Effectuer les modifications demandées.

### 4. Ajouter les fichiers modifiés

```bash
git add .
```

### 5. Créer un commit

```bash
git commit -m "feat: ajout de la barre de navigation"
```

### 6. Envoyer la branche sur GitHub

```bash
git push -u origin feature/nom-de-la-tache
```

### 7. Créer une Pull Request

Depuis GitHub :

- Pull Requests
- New Pull Request
- Choisir :
  - Base : `main`
  - Compare : votre branche

### 8. Attendre la revue du code

Une autre personne du groupe doit relire la Pull Request avant validation.

### 9. Fusion (Merge)

Une fois approuvée :

- Merge Pull Request
- Delete Branch
